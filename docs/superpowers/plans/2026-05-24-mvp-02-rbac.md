# IFP Connect MVP — Plano 2: RBAC + Audit Log + RLS — Implementation Plan

> **Status:** Decisões §0.1, §0.2, §0.5, §0.9 **FECHADAS em 2026-05-24** após conversa com Erick. Decisões §0.3, §0.4, §0.6, §0.7, §0.8 ainda abertas — implementação pode começar pelas tarefas que dependem só das decisões fechadas.

**Goal:** Adicionar controle de acesso baseado em papéis (RBAC), audit log de eventos sensíveis, Row-Level Security (RLS) no Postgres, e dashboards específicos por contexto.

**Architecture:** Modelo many-to-many `User <-> Role` (com `unitScope` opcional). Permissões derivadas do role (capability-based access control). Audit log como tabela append-only. RLS no Postgres via policies SQL que leem `app.current_user_id` setado pelo Prisma. **3 dashboards base** (Global / Social / Unidade) servidos via rotas role-aware.

**Tech Stack adicional:** `pgTAP` para testes de RLS, helper `withAuthContext()` para setar `app.current_user_id` no início de cada Prisma transaction.

**Plano base:** `docs/superpowers/plans/2026-05-23-ifp-connect-mvp-01-foundation.md` (✅ done)
