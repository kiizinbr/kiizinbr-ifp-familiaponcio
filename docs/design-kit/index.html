Design Kit · IFP Connect

A fundação visual de tudo que vem depois.

Este kit não desenha telas — ele desenha como qualquer tela futura será montada. Tokens, componentes e esqueletos de página prontos para forkar, na direção "clínica premium" validada no mockup do prontuário. Denso e preciso para a equipe; digno e acolhedor para o cidadão. Uma marca só (IFP); a cor da unidade entra como sinal funcional discreto — experimente o seletor de unidade ali em cima.

1 · Copie a fundação

ifp-tokens.css + ifp-components.css vão para src/styles/ e são importados no globals.css. Tokens são canônicos do brandbook.

2 · Forke um scaffold

Cada esqueleto de página é um HTML autossuficiente. Abra, copie a estrutura, troque o conteúdo. App shell, 3 colunas, lista, dashboard, wizard, login.

3 · Aplique o sotaque

data-unit="esportivo" no body já reorienta faixas e selos para a cor da unidade — sem virar 4 apps diferentes.

A direção

Clínica premium, não portal de governo.

✓ Buscar

Densidade alta com hierarquia clara · cor como sinal funcional · tipografia afiada (Hanken Grotesk + IBM Plex Mono nos dados) · sensação Doctolib / Elation / Linear · dignidade e confiança.

✕ Evitar

Tema "Editorial" abandonado (serifa Fraunces, fundo creme) · cara de sistema público antigo · 4 marcas diferentes por unidade · dark patterns no consentimento (público vulnerável).

Padrões · decisões

Regras que fazem o sistema parecer um só

Cor como sinal funcional

App autenticado = 1 marca IFP. A cor da unidade entra como faixa de topo, selo ou slot de agenda — nunca repinta a tela inteira. A "explosão de cor" fica no login.

Densidade vs. acolhimento

Telas de equipe (agenda, fila, prontuário, tabelas) = densidade alta, info por pixel. Telas de cidadão/família (login, funil, certificado, feed) = respiro, escala maior, calor.

Uso do leão

SIM: login, landing, empty state (30%), certificado (grande), 404/500, boas-vindas. NÃO: sidebar (só o símbolo a 32px), headers operacionais, modais, toasts.

Empty states por papel

A mensagem muda com quem está logado: "Sem triagens pendentes" (Regina/social) ≠ "Nenhuma vaga aberta" (recepção) ≠ "Você ainda não tem cidadãos" (gestor).

Atos clínicos pessoais

Editar/assinar nota não tem bypass de admin. Visualmente: o botão "Assinar" só aparece ativo para o dono da nota; para os outros, fica inerte com tooltip.

Consentimento honesto

Público vulnerável. Nada de dark pattern: checkbox de consentimento sempre desmarcado, linguagem clara, sem CPF/dados sensíveis na rota pública.

● Para telas ainda não idealizadas: comece sempre pelo App Shell, escolha o scaffold mais próximo do tipo de tarefa (lista, fluxo, foco, dashboard), aplique data-unit e siga as 6 regras acima. O resto sai dos componentes.
